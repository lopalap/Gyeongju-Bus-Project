[최종 보고서] Gyeongju Bus Live: 실시간 위치 기반 지능형 알림 시스템
1. 프로젝트 개요
프로젝트명: Gyeongju Bus Live (경주 실시간 버스 정보 서비스)

개발 배경: 기존 지도 서비스는 사용자가 직접 검색하고 확인해야 하는 수동적인 방식입니다. 본 프로젝트는 공공 데이터를 가공하여 사용자에게 도착 3분 전 능동적으로 푸시 알림을 제공함으로써, 실생활의 불편함을 기술적으로 해결하고자 기획되었습니다.

목표: React와 Node.js를 활용한 풀스택 시스템 구축 및 실시간 데이터 처리 자동화.

2. 개발 환경 및 기술 스택
[Software Stack]
Frontend: React.js (Component 기반 UI 설계), Axios (비동기 통신)

Backend: Node.js, Express (RESTful API 서버 구축)

Database: MariaDB (사용자 즐겨찾기 및 정류장 메타데이터 관리)

External API: 공공데이터포털(국토교통부 버스 도착 정보), Kakao Maps SDK

3. 핵심 구현 내용 (구현 상세 및 차별점)
1) 실시간 데이터 파이프라인 구축 (Backend)
직접 구현: 공공데이터 API의 XML/JSON 데이터를 Node.js 서버에서 수신하여 필요한 정보(노선 번호, 남은 시간, 현재 위치)만 추출하는 데이터 정제 로직을 직접 개발했습니다.

차별점: 클라이언트가 외부 API에 직접 요청하는 대신, 자체 API 서버를 거치게 하여 데이터 보안을 강화하고 프론트엔드의 연산 부하를 줄였습니다.

2) 지능형 실시간 알림 시스템 (Web Notification API)
알림 로직:

서버에서 수신한 잔여 시간이 180초(3분) 미만이 되는 시점을 실시간 감지.

브라우저의 Web Notification API를 호출하여 사용자 화면 하단에 즉시 푸시 알림 전송.

기술적 성과: 브라우저가 최소화된 상태에서도 작동하도록 설계하여, 사용자가 앱을 계속 들여다보지 않아도 되는 **능동적 UX(User Experience)**를 구현했습니다.

3) 동적 지도 인터랙션 (Kakao Maps SDK)
커스터마이징: 카카오맵을 단순 표시하는 것을 넘어, 서버에서 받은 정류장 위도/경도 좌표를 이용해 **커스텀 마커(Overlay)**를 동적으로 생성합니다.

인터페이스 연동: 마커 클릭 시 MariaDB의 정류장 정보와 실시간 도착 정보를 결합한 **인포윈도우(InfoWindow)**를 노출하여 정보 접근성을 극대화했습니다.

4) 실시간 데이터 동기화 및 개인화
Live Update: React의 useEffect와 setInterval을 활용하여 1초 단위로 잔여 시간을 업데이트하는 실시간 카운트다운 기능을 구현했습니다.

즐겨찾기 시스템: 사용자가 즐겨찾는 노선을 DB에 저장하고 관리하는 기능을 통해 개인화된 서비스를 제공합니다.

4. 프로젝트 추진 일정 (4주간의 성과)
1주차 (기획/설계): 요구사항 정의서 작성 및 DB E-R 다이어그램 설계.

2주차 (서버 개발): Node.js 서버 구축 및 공공 API 연동 테스트 완료.

3주차 (프론트엔드): React UI 컴포넌트 개발 및 지도 API 연동.

4주차 (고도화): Web Notification API를 활용한 알림 로직 통합 및 전체 테스트 완료.

결론 및 향후 과제
본 프로젝트는 기존 플랫폼(카카오맵)을 단순 활용하는 것을 넘어, 공공 데이터의 실시간 가공과 자동 알림 시스템을 직접 설계하는 풀스택 개발 역량을 보여주는 데 집중했습니다. 이를 통해 복합적인 API를 하나의 완성된 서비스로 통합하는 능력을 함양할 수 있었습니다.
