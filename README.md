[최종보고서] 경주 실시간 버스 정보 서비스 (Gyeongju Bus Live)
1. 프로젝트 개요
프로젝트명: Gyeongju Bus Live (경주 실시간 버스 정보 시스템)

개발 목적: 공공 데이터를 활용하여 경주 지역 사용자가 버스 도착 정보를 실시간으로 확인하고, 브라우저 푸시 알림을 통해 버스를 놓치지 않도록 돕는 개인 맞춤형 웹 서비스 구축.

주요 특징: React 기반의 SPA(Single Page Application)와 Node.js 백엔드 서버를 결합한 풀스택 시스템.

2. 시스템 구성 및 기술 스택
🛠 Technology Stack
Frontend: React.js, Kakao Maps SDK, Web Notification API

Backend: Node.js, Express (REST API 서버 직접 설계)

Database: MariaDB (정류장 정보 및 사용자 즐겨찾기 저장)

⚙ 시스템 아키텍처
공공데이터 포털 API: 경주시 실시간 버스 데이터 수집.

Node.js Server: 외부 API 데이터를 정제하고 클라이언트에 전달하는 자체 API 서버 구축.

React Frontend: 지도 시각화 및 실시간 알림 UI 제공.

3. 핵심 기능 및 구현 상세 (기술적 차별점)
📍 1) 실시간 데이터 시각화 (지도 연동)
단순히 지도를 보여주는 것에 그치지 않고, 공공 API의 좌표 데이터를 활용해 지도 위에 실시간 마커를 렌더링하는 로직을 직접 구현했습니다.

정류장 클릭 시 DB와 통신하여 해당 위치의 실시간 도착 정보를 팝업으로 노출합니다.

🔔 2) 지능형 푸시 알림 (Web Notification API) - 독창적 핵심 기능
기존 서비스와의 차별점: 기성 지도 앱과 달리, 사용자가 웹 브라우저를 백그라운드에 두어도 도착 3분 전에 자동으로 푸시 알림이 발송되는 자동화 로직을 직접 설계했습니다.

이 기능은 사용자가 일일이 화면을 확인하지 않아도 되도록 하여 UX(사용자 경험)를 고도화했습니다.

⚡ 3) 실시간 동기화 및 개인화
Live Count-down: 프론트엔드에서 1초 단위로 잔여 시간을 실시간 연산하여 동적인 데이터를 제공합니다.

즐겨찾기: MariaDB 연동을 통해 자주 가는 정류장을 메인 화면에서 바로 확인하도록 개인화 기능을 추가했습니다.

4. 개발 일정 준수 현황
1주차: 요구사항 분석, API 선정 및 환경 구축 완료

2주차: Node.js 서버 및 DB 스키마 설계 완료

3주차: React UI 개발 및 실시간 알림 시스템 연동 완료

4주차: 최종 통합 테스트 및 리포지토리 배포 완료

5. 결론 및 기대효과
본 프로젝트는 기존 지도를 인터페이스(UI) 플랫폼으로 활용하여, 실시간 데이터를 어떻게 가공하고 사용자에게 능동적으로 전달할 수 있는지를 탐구한 결과물입니다. 특히 자체 API 서버와 알림 시스템을 직접 구축함으로써 풀스택 웹 개발자로서의 역량을 증명하고자 했습니다
